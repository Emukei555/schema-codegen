# 0001. Adoption of C ABI-Based Primitive Type Decomposition for Cross-Language FFI

**Date:** 2026-07-21
**Status:** Accepted

## Context
In this schema compiler (`schema-codegen`), it is ultimately necessary to establish a foundation that enables seamless type conversion and data communication between Rust and C++.
However, Rust and C++ each possess their own distinct ABI (Application Binary Interface), making their memory layouts and object management methods (e.g., name mangling, exception handling, virtual method tables) incompatible. Consequently, sharing generated C++ structures and Rust structures directly in the same memory space is a highly dangerous approach that can lead to Undefined Behavior (UB).

It was necessary to formulate a design strategy to achieve safe and reliable cross-language communication.

## Decision
We avoid direct communication between Rust and C++ and adopt the strategy of **decomposing complex types into C ABI-based primitive types for communication (C ABI Binding).**

Specifically, we will extend the code generator (Codegen) according to the following rules:
1. **Primitive Decomposition:** Complex types such as structs (Objects), arrays (Vectors), and strings (Strings) will be decomposed into C primitive types, such as pointers and sizes (e.g., `const uint8_t* data`, `size_t len`), before being passed across the boundary.
2. **Utilization of `extern "C"`:** Generate FFI boundary functions using `extern "C"` on the Rust side, and generate wrapper functions on the C++ side to invoke them.
3. **State Validity at the Boundary:** Data crossing the FFI boundary is assumed to already be in a valid state (Always-Valid) that satisfies the constraints defined on the Parser/IR side (e.g., the prohibition of multi-dimensional arrays).

## Consequences

### Good
* **High Stability and Compatibility:** The C ABI is the de facto standard, providing a stable communication foundation unaffected by changes in compiler versions or language specifications.
* **Clear Boundary Separation:** The FFI layer clearly functions as a boundary for data conversion, successfully decoupling the communication logic from the domain logic (the internal implementations of Rust and C++).
* **Improved Memory Safety:** Safety is easier to guarantee by treating data as explicit primitive copies or references, avoiding the cross-boundary management of complex object ownership and lifetimes.

### Bad
* **Increased Boilerplate:** Glue code (wrapper functions) required to mediate through the C ABI will be necessary on both language sides.
    * *Mitigation:* Because all of this code is automatically generated by `schema-codegen`, there will be no manual maintenance cost for developers.
* **Minor Performance Overhead:** Compared to direct memory casting, the steps of primitive decomposition and reconstruction introduce a potential for slight overhead.
